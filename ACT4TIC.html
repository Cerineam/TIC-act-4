    Chapitre 3 :Le modèle OSI

    fait par:BENMOUSSA Manil & GHEDAIFI Ahmed

    1)Les Composants matériel d’un Réseau informatique:

    -Un réseau informatique est composé de plusieurs éléments matériels qui permettent aux ordinateurs de communiquer entre eux.

    1.Les cartes réseau ou NIC:

    -sont des composants installés à l'intérieur des ordinateurs et permettent de se connecter à un réseau en utilisant des protocoles de communication spécifiques tels que TCP/IP.

    2.Les switches:

    -permettent de connecter plusieurs appareils sur un réseau local en utilisant des adresses MAC et en stockant ces informations dans une table de correspondance. Les switches peuvent être utilisés dans des réseaux de différentes tailles, offrant des fonctionnalités avancées.

    3.Les hubs:

    -quant à eux, fonctionnent comme des répéteurs en transmettant les données à tous les appareils connectés, mais sans utiliser de table de correspondance pour envoyer les données uniquement à l'appareil destinataire. Les hubs ont été largement remplacés par les switches pour leur efficacité dans la gestion du trafic réseau et pour permettre une communication plus rapide et plus sécurisée entre les appareils connectés.

    2)Le Modèle OSI:

    -Le modèle OSI est un modèle de référence pour les communications de réseau, qui divise les communications en sept couches distinctes. Chaque couche est responsable de fonctions spécifiques liées à la communication de données entre les appareils de réseau. Les sept couches du modèle OSI sont : physique, liaison de données, réseau, transport, session, présentation et application. Bien que le modèle OSI ne soit pas utilisé dans les protocoles de communication modernes tels que TCP/IP, il reste utile pour comprendre les principes fondamentaux de la communication de réseau.

    3) Le Modèle TCP/IP:

    -Le modèle TCP/IP est un modèle de référence pour les communications de réseau. Il est utilisé pour la conception et la mise en œuvre des réseaux informatiques modernes et se compose de quatre couches principales:

  • la couche liaison de données.
  • la couche réseau.
  • la couche transport.
  • la couche application.

    • -Contrairement au modèle OSI, qui comprend sept couches distinctes, le modèle TCP/IP est le modèle de référence pour les protocoles de communication de réseau les plus largement utilisés,tels que:

  • le protocole Internet.
  • le protocole de contrôle de transmission.
  • le protocole de datagramme utilisateur.

    • -Le modèle TCP/IP est utilisé pour la conception et la mise en œuvre des réseaux d'entreprise, des réseaux Internet et des réseaux sans fil.

    4)L’adressage:

    L'adressage est un élément essentiel dans la communication en réseau. Il permet d'identifier de manière unique chaque unité participant à la communication. Dans ce cadre,C’est quoi un adressage IP.

    1)L’adressage IP:

    l'adresse IP est un numéro qui identifie chaque ordinateur dans les réseaux auxquels il appartient. Il existe deux modes d'adressage IP : IPv4 et IPv6.

    1.IPv4:

    -Dans le cas d'IPv4, chaque paquet de données contient une adresse source sur 32 bits et une adresse destinataire sur 32 bits. Ainsi, l'adresse IPv4 est représentée sur 4 octets, soit 32 bits.

    2.IPv6:

    -Dans le cas d'IPv6, les adresses sont codées sur 128 bits. La représentation textuelle se fait en découpant le mot de 128 bits de l'adresse en 8 mots de 16 bits séparés par le caractère «:», chacun d'eux est en hexadécimal.

    5)L'adresse IPV4:

  • -L'adresse IPV4 est une adresse numérique qui est utilisée pour identifier un périphérique dans un réseau. Cette adresse est constituée de 32 bits divisés en 4 octets et est généralement représentée sous forme de quatre nombres séparés par des points. Les deux parties d'une adresse IPV4 sont: l'adresse réseau et l'adresse hôte.

    • 1.L'adresse réseau:

    -est la partie des bits les plus significatifs qui identifie le réseau auquel l'appareil appartient.

    2.L'adresse hôte:

    -est la partie des bits restants qui identifie l'appareil sur le réseau.

    pour résumé,On peut identifier un réseau à partir de la partie réseau de l'adresse en vérifiant si la valeur des bits est la même.

  • Il existe différents types d'adresses IPV4:

    • 1)L'adresse réseau:

    -L'adresse réseau est une adresse où tous les bits de la partie hôte sont égaux à zéro.

    2)L'adresse de diffusion:

    -L'adresse de diffusion est une adresse où tous les bits de la partie hôte sont égaux à un. Elle est utilisée pour envoyer des données à tous les hôtes sur un réseau

    3)Les adresses d'hôtes:

    -Les adresses d'hôtes sont les adresses situées entre l'adresse réseau et l'adresse de diffusion et sont attribuées aux périphériques sur le réseau.

    4)Le masque de réseau et de sous-réseau:

    -Le masque de réseau et de sous-réseau est utilisé pour déterminer l'adresse réseau et l'adresse hôte à partir d'une adresse IP. Il est généralement représenté sous forme de quatre nombres séparés par des points, tout comme l'adresse IP. Le masque de réseau et de sous-réseau est utilisé pour diviser un réseau en sous-réseaux plus petits, ce qui permet de mieux gérer et de contrôler le trafic sur le réseau.

    1)Qu’est-ce que Git ?

    -Git est un logiciel de versioning qui permet de conserver un historique des modifications effectuées sur un projet afin de pouvoir identifier les changements effectués, il facilite la gestion des projets de manière efficace.

    2)à quoi sert concrètement un système de gestion de version?

    -Git permet à différentes versions d’un même fichier de coexister et permet de coordonner le travail entre plusieurs personnes (accès à l’historique des modifications pour l’intégralité savoir quels changements ont été fait par rapport à leur version des fichiers.

    3) Les deux modèles des logiciels de gestion de version : modèle centralisé VS modèle décentralisé

    1.Le modèle centralisé:

    -la source du code du projet est hébergée sur un serveur distant central et les différents utilisateurs doivent se connecter à ce serveur pour travailler sur ce code.

    2.Le modèle distribué (décentralisé):

    -chaque utilisateur est invité à télécharger et à héberger l’intégralité du code source du projet sur sa propre machine. Ses aventages par rapport au modèle centralisé est la Simplicité / flexibilité du travail et la Sécurité.

    4) Qu’est-ce que GitHub?

    -Dans le langage des systèmes de gestion de version, la copie de l’intégralité des fichiers d’un projet et de leur version située sur le serveur central est appelé un dépôt. GitHub est un service en ligne qui permet d’héberger des dépôts ou repo Git.

    5)comment Utiliser Git ?:ligne de commande, console et interface graphique.

    -Utiliser la ligne de commande est le seul moyen d’avoir accès à toutes les commandes Git.Pour ouvrir la ligne de commande, rechercher cmd dans la barre de recherche.

    • 8)Démarrer un dépôt Git

    -Il existe deux façons de créer un dépôt Git :

  • On peut importer un répertoire déjà existant dans Git
  • On peut cloner un dépôt Git déjà existant.

    • 9)La gestion des informations selon Git:

    -Grosso modo, à chaque fois qu’on va valider ou enregistrer l’état d’un projet dans Git, il va prendre un instantané du contenu de l’espace de travail à ce moment et va enregistrer une référence à cet instantané pour qu’on puisse y accéder par la suite.Chaque instantané est stocké dans une base de données locale.

    10)Les états des fichiers:

    -suivi et non suivi. Un fichier est suivi s'il appartient à la dernière capture instantanée effectuée par Git, et il est non suivi s'il n'était pas inclus dans la dernière capture instantanée et n'a pas été indexé. Les fichiers peuvent avoir trois étapes : modifié, préparé et validé. Lorsqu'on démarre un référentiel Git, tous les fichiers sont non suivis et doivent être indexés et validés. Les fichiers peuvent être modifiés ou nouveaux, et ils sont suivis lorsqu'ils sont indexés. Enfin, lorsqu'on prend une capture instantanée, les fichiers sont validés, et le cycle recommence.

    11)Les zones de travail:

    -les trois zones de travail dans Git : le répertoire de travail (working tree), la zone d'index (staging area) et le répertoire Git (repository). Le répertoire de travail contient les fichiers extraits de la base de données compressée située dans le répertoire Git et permet de les modifier. La zone d'index stocke les informations sur ce qui fera partie du prochain instantané ou du prochain "commit". Le répertoire Git est l'endroit où Git stocke les méta-données et la base de données des objets de votre projet. Le processus de travail consiste à travailler sur les fichiers dans le répertoire de travail, indexer les fichiers modifiés ou nouveaux dans la zone d'index, et enfin valider la version indexée pour l'ajouter au répertoire Git.

    12)comment Créer un dépôt Git à partir d’un répertoire existant?

    1.-créer un dépôt Git à partir d'un répertoire existant contenant des projets stockés sur une machine ou un serveur distant. Il donne des commandes utiles telles que "cd" pour se positionner dans un répertoire et "dir" pour afficher la liste des dossiers et fichiers dans l’emplacement actuel

    2.-initialiser un dépôt Git et indexer des fichiers afin de les versionner. La commande "git init" permet d'initialiser un dépôt Git vide. La commande "git status" permet de vérifier l'état des fichiers du répertoire, et la commande "git add" permet d'ajouter des fichiers à l'index. Les "fileglobs" (utilisant des caractères spéciaux comme "*" et "?") permettent de sélectionner des fichiers en fonction de leur extension. Ensuite, les fichiers peuvent être validés ("commit") pour être ajoutés à la base de données Git.

    -Ce texte explique comment utiliser Git pour indexer et versionner des fichiers. La commande "git init" permet d'initialiser un dépôt Git vide, tandis que "git status" permet de vérifier l'état des fichiers. "git add" sert à indexer des fichiers déjà sous suivi de version et à placer sous suivi des fichiers non suivi. Ensuite, les fichiers peuvent être validés ("commit") pour être ajoutés à la base de données Git. Il est important de bien documenter chaque commit pour permettre aux contributeurs de comprendre les modifications et de les valider. Enfin, le texte explique comment cloner un dépôt Git et comment vérifier que tous les fichiers sont sous suivi de version et enregistrés en base.

    13)Cloner un dépôt Git:

    -La commande "Git clone" permet de cloner localement un dépôt Git existant. C'est une autre façon de démarrer un dépôt Git.

    14)Ajouter ou modifier des fichiers dans un projet et actualiser notre dépôt Git:

    -Il est important de signaler à Git tous les changements apportés pour que l'historique des versions puisse être conservé. Pour enregistrer une modification ou un ajout de fichier dans le dépôt Git, il faut utiliser les commandes git add et git commit, en veillant à bien effectuer un git add avant chaque git commit pour enregistrer la dernière version du fichier. Enfin, pour mettre en index les fichiers modifiés et déjà sous suivi, il est possible d'utiliser la commande "git add ." pour ensuite les valider avec git commit.

    15)Consulter l’historique des modifications Git:

    -La commande "git log" permet de voir l'historique des modifications Git, avec la liste des commits réalisés du plus récent au plus ancien, affichant par défaut la somme de contrôle SHA-1, le nom et l'e-mail de l'auteur, la date et le message du commit.

    16)Annuler des modifications apportées à un fichier:

    -Un logiciel de gestion de version permet de revenir à un état antérieur d'un projet enregistré, ce qui est l'un de ses principaux avantages. Après un commit, si des modifications ne donnent pas les résultats escomptés, il est possible de revenir à l'état précédent du fichier en utilisant la commande "git checkout -- nom-du-fichier" ou la nouvelle commande "git restore".

    17)Qu’est ce qu’une branche ?:

    -Git permet de créer des branches facilement en utilisant un simple pointeur vers un commit. Contrairement à d'autres systèmes de contrôle de version, Git ne copie pas l'intégralité du répertoire de travail, ce qui facilite la création et la fusion de branches. La branche par défaut dans Git est appelée "master" et elle se déplace automatiquement à chaque nouveau commit. En résumé, une branche est simplement un pointeur vers un commit en particulier.
